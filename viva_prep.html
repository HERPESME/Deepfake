1. Project Overview

Objective: Develop and compare deep learning models for detecting deepfake images and videos.

Approach: Train and evaluate four different architectures (EfficientNet-B0, XceptionNet, ResNet50, Vision Transformer) on Celeb-DF dataset, then test generalization on FaceForensics++.

Key Innovation: Cross-dataset evaluation to assess model robustness and generalization capabilities.

Project Pipeline

  1. Data Preprocessing: Extract frames from videos → Detect faces → Crop and resize → Normalize
  2. Model Training: Train 4 models on Celeb-DF with optimized hyperparameters
  3. In-Distribution Evaluation: Test on Celeb-DF test set
  4. Cross-Dataset Evaluation: Test on FaceForensics++ (different distribution)
  5. Explainability Analysis: Generate Grad-CAM visualizations to understand model decisions

2. Data Preprocessing Pipeline

The preprocessing pipeline converts raw video files into standardized face images ready for model training.

2.1 Frame Extraction

Code Location: src/data/preprocessing.py - VideoProcessor.extract_frames()

How it works:

  • Open video file using OpenCV's VideoCapture
  • Extract frames at regular intervals (every 30th frame for Celeb-DF, every 60th for FaceForensics++)
  • Limit frames per video (max 10) to balance dataset size and diversity
  • Save frames as temporary image files

2.2 Face Detection

Code Location: src/data/preprocessing.py - FaceDetector.detect_faces()

Algorithm: OpenCV Haar Cascade Classifier

How it works:

  • Convert image to grayscale (face detection works on grayscale)
  • Apply Haar Cascade classifier: Uses pre-trained features (edges, lines) to detect face patterns
  • Returns bounding boxes (x, y, width, height) for detected faces
  • Select largest face if multiple faces detected

2.3 Face Cropping & Resizing

Code Location: src/data/preprocessing.py - FaceDetector.crop_face()

Steps:

  1. Add 20% padding around detected face (preserves context)
  2. Crop face region from original image
  3. Resize to 224×224 pixels (standard input size for all models)
  4. Normalize pixel values to [0, 1] range

2.4 Data Splitting

Code Location: src/data/preprocessing.py - DatasetPreprocessor.create_splits()

Split Ratio: 80% Training, 10% Validation, 10% Test

Method: Random split with fixed seed (42) for reproducibility

3. Model Architectures Explained

All models follow a similar structure: Backbone (Feature Extractor) → Classifier Head

3.1 EfficientNet-B0

Code Location: src/models/baseline_models.py - EfficientNetModel

Parameters: 4.6 million

Key Innovation: Compound Scaling - simultaneously scales depth, width, and resolution

Architecture:

  • Backbone: EfficientNet-B0 (pretrained on ImageNet) extracts 1280-dimensional features
  • Classifier: Dropout(0.5) → Linear(1280 → 512) → ReLU → Dropout(0.5) → Linear(512 → 2)
  • Output: 2 logits (one for Real, one for Fake)

3.2 XceptionNet

Code Location: src/models/baseline_models.py - XceptionNet

Parameters: 38.9 million

Key Innovation: Depthwise Separable Convolutions - reduces computation while maintaining capacity

How Depthwise Separable Convolution Works:

  1. Depthwise Convolution: Each input channel is convolved separately (spatial filtering)
  2. Pointwise Convolution: 1×1 convolution combines channels (channel mixing)

Result: Similar representational power with ~8× fewer parameters than standard convolution

Classifier: Same structure as EfficientNet (Dropout → Linear → ReLU → Dropout → Linear)

3.3 ResNet50

Code Location: src/models/baseline_models.py - ResNetModel

Parameters: 25 million

Key Innovation: Residual Connections (Skip Connections)

How Residual Blocks Work:

  • Each block computes: output = F(x) + x where F(x) is the learned transformation
  • Why it works: Allows gradients to flow directly through skip connections, enabling training of very deep networks (50+ layers)
  • Architecture: 4 residual blocks (layer1-4) with increasing channels (64→128→256→512)
  • Classifier: Global Average Pooling → Dropout → Linear(2048 → 512) → ReLU → Dropout → Linear(512 → 2)

3.4 Vision Transformer (ViT)

Code Location: src/models/baseline_models.py - VisionTransformer

Parameters: 86 million

Key Innovation: Treats images as sequences of patches, uses self-attention instead of convolutions

How ViT Works (Step-by-Step):

  1. Patch Embedding: Split 224×224 image into 16×16 patches (196 patches total)
    • Each patch (16×16×3 = 768 pixels) → Linear projection to 768 dimensions
  2. Add Class Token: Prepend learnable [CLS] token (for classification)
  3. Positional Embedding: Add learnable position encodings (tells model where each patch is)
  4. Transformer Encoder (12 layers):
    • Multi-Head Self-Attention: Each patch attends to all other patches
    • MLP (Feed-Forward): Two linear layers with GELU activation
    • Layer Normalization: Applied before each sub-layer
  5. Classification: Use [CLS] token output → Linear(768 → 512) → ReLU → Linear(512 → 2)
Self-Attention Formula: Attention(Q, K, V) = softmax(QK^T / √d_k) × V
  • Q (Query), K (Key), V (Value) are learned linear projections
  • Computes how much each patch should attend to every other patch

4. Training Process & Algorithms

Code Location: src/training/trainer.py - DeepfakeTrainer.train()

4.1 Training Loop Structure

For each epoch:

  1. Training Phase: Forward pass → Loss calculation → Backward pass → Weight update
  2. Validation Phase: Forward pass only (no gradient computation) → Calculate metrics
  3. Learning Rate Scheduling: Adjust learning rate based on schedule
  4. Early Stopping Check: Stop if validation performance doesn't improve
  5. Checkpoint Saving: Save model every N epochs

4.2 Loss Function: Cross-Entropy Loss

Formula: L = -[y × log(ŷ) + (1-y) × log(1-ŷ)]
  • y: True label (0 for Real, 1 for Fake)
  • ŷ: Predicted probability of Fake class (from softmax)
  • Why it works: Penalizes confident wrong predictions heavily, encourages correct predictions
  • Implementation: nn.CrossEntropyLoss() in PyTorch (combines LogSoftmax + NLLLoss)

4.3 Optimizers

4.3.1 Adam Optimizer

Used for: EfficientNet, XceptionNet, ViT

How it works:

  1. Maintains exponential moving averages of gradients (m_t) and squared gradients (v_t)
  2. Bias correction: Adjusts for initialization bias
  3. Adaptive learning rate: Each parameter gets its own learning rate based on gradient history

Advantages: Fast convergence, works well with default hyperparameters

Hyperparameters: β₁=0.9 (momentum decay), β₂=0.999 (squared gradient decay), ε=10⁻⁸

4.3.2 AdamW Optimizer

Used for: ResNet50

Difference from Adam: Decouples weight decay from gradient-based updates

Why better: More effective regularization, better generalization

Formula: θ_{t+1} = θ_t - η × (m̂_t / (√v̂_t + ε)) - λ × η × θ_t

(last term is decoupled weight decay)

4.4 Learning Rate Schedulers

4.4.1 Cosine Annealing

Used for: EfficientNet, XceptionNet, ViT

Formula: η_t = η_min + (η_max - η_min) × (1 + cos(πt/T)) / 2
  • How it works: Learning rate decreases smoothly following a cosine curve
  • Advantages: Smooth decay, helps fine-tune in later epochs

4.4.2 ReduceLROnPlateau

Used for: ResNet50

How it works:

  • Monitors validation AUC (Area Under ROC Curve)
  • If validation AUC doesn't improve for 'patience' epochs → reduce LR by factor (e.g., 0.5)
  • Advantages: Adaptive - only reduces LR when needed

4.5 Early Stopping

Code Location: src/training/trainer.py - EarlyStopping

How it works:

  • Tracks best validation AUC score
  • If validation AUC doesn't improve for 'patience' epochs (10-20) → stop training
  • Restores best model weights before stopping
  • Purpose: Prevents overfitting, saves computation time

4.6 Mixed Precision Training

What it is: Uses FP16 (half precision) for forward/backward passes, FP32 for weight updates

  • Benefits: 30-50% faster training, 50% less memory usage
  • Implementation: PyTorch's torch.cuda.amp with GradScaler

5. Evaluation Metrics & Interpretation

Code Location: src/evaluation/metrics.py - MetricsCalculator

5.1 Confusion Matrix

Predicted Real Predicted Fake
Actual Real TN (True Negative) FP (False Positive)
Actual Fake FN (False Negative) TP (True Positive)

5.2 Classification Metrics

Accuracy: (TP + TN) / (TP + TN + FP + FN)

  • Interpretation: Overall percentage of correct predictions
  • Limitation: Can be misleading with imbalanced datasets

Precision: TP / (TP + FP)

  • Interpretation: Of all predictions labeled 'Fake', how many are actually fake?
  • High Precision: When model says 'Fake', it's usually correct (few false alarms)

Recall (Sensitivity): TP / (TP + FN)

  • Interpretation: Of all actual fakes, how many did we catch?
  • High Recall: Model finds most fakes (few missed detections)

F1-Score: 2 × (Precision × Recall) / (Precision + Recall)

  • Interpretation: Harmonic mean of Precision and Recall
  • Use case: Single metric balancing precision and recall

5.3 ROC Curve & AUC

ROC Curve: Plots True Positive Rate (TPR) vs False Positive Rate (FPR) at different thresholds

  • TPR (Sensitivity): TP / (TP + FN) - How many fakes we catch
  • FPR: FP / (FP + TN) - How many reals we incorrectly flag as fake
  • AUC (Area Under Curve): Measures overall discriminative ability
    • AUC = 1.0: Perfect classifier
    • AUC = 0.5: Random guessing
    • AUC > 0.9: Excellent performance
  • Why useful: Threshold-independent metric, works well with imbalanced data

5.4 Precision-Recall Curve

What it shows: Trade-off between Precision and Recall at different thresholds

  • Better than ROC for imbalanced datasets (we have more fake than real images)
  • High curve: Can achieve high precision and recall simultaneously

8. Key Algorithms Explained

8.1 Backpropagation (How Models Learn)

Process:

  1. Forward Pass: Input → Model → Output (predictions)
  2. Loss Calculation: Compare predictions with true labels
  3. Backward Pass: Compute gradients (derivatives) of loss w.r.t. each parameter
  4. Gradient Descent: Update parameters: θ = θ - η × ∇L(θ)
    • η (eta) = learning rate, ∇L(θ) = gradient of loss
  5. Repeat: Thousands of iterations until loss is minimized

8.2 Gradient Calculation (Chain Rule)

For a neural network: L = loss(y, f(x; θ))

  • Compute ∂L/∂θ using chain rule: ∂L/∂θ = (∂L/∂ŷ) × (∂ŷ/∂θ)
  • PyTorch's autograd automatically computes these gradients
  • loss.backward() triggers backpropagation through entire network

8.3 Batch Processing

Why batches?

  • Computing gradients on entire dataset is too slow
  • Single sample has high variance
  • Solution: Process 32 samples at a time (batch_size=32)
  • Average gradients over batch → more stable updates

8.4 Dropout Regularization

What it does: Randomly sets 50% of neurons to zero during training

  • Why: Prevents overfitting by forcing model to not rely on specific neurons
  • During inference: All neurons active, but outputs scaled by dropout probability
  • Implementation: nn.Dropout(0.5) in classifier head

8.5 Softmax Activation

Formula: softmax(x_i) = exp(x_i) / Σ exp(x_j)
  • Converts raw logits to probabilities (sums to 1.0)
  • Example: logits = [2.0, 1.0] → probabilities = [0.73, 0.27]
  • Higher logit → higher probability

9. Common Viva Questions & Answers

Q: Why did you choose these four models?

A: EfficientNet for efficiency, XceptionNet as proven baseline, ResNet50 for residual learning, ViT to explore transformer-based approach. This gives diverse architectural perspectives.

Q: Why does ViT have lower in-distribution accuracy?

A: ViT requires more data to train effectively. With limited training data, CNNs' inductive biases (translation equivariance, locality) help them learn better. ViT's attention mechanism needs more examples to learn spatial relationships.

Q: Why do models fail on cross-dataset evaluation?

A: Models learn dataset-specific features (compression artifacts, video quality, generation method characteristics) rather than universal deepfake patterns. Different datasets have different distributions, causing domain shift.

Q: Why does ViT generalize better?

A: Self-attention mechanism allows ViT to learn global relationships between image patches, potentially capturing more generalizable features. CNNs' local receptive fields may overfit to local patterns specific to training data.

Q: What is the difference between accuracy and AUC?

A: Accuracy measures correctness at a fixed threshold (usually 0.5). AUC measures discriminative ability across all possible thresholds. AUC is better for imbalanced datasets and threshold-independent evaluation.

Q: How does Grad-CAM work?

A: Grad-CAM computes gradients of target class score w.r.t. feature maps, then weights feature maps by these gradients. This highlights image regions most important for the prediction. Red regions = high importance.

Q: What is early stopping and why use it?

A: Early stopping monitors validation performance and stops training when it stops improving. Prevents overfitting (model memorizing training data) and saves computation time.

Q: What is the purpose of learning rate scheduling?

A: Start with higher LR for fast learning, gradually reduce for fine-tuning. Cosine annealing provides smooth decay, while plateau-based reduces only when needed.

Q: How do you handle class imbalance?

A: We use weighted sampling in DataLoader, but our dataset is relatively balanced. For severe imbalance, we could use focal loss or class weights in cross-entropy loss.

Q: What improvements would you make?

A: 1) Multi-dataset training for better generalization, 2) Ensemble methods combining CNN and ViT, 3) Temporal analysis for video sequences, 4) Adversarial training for robustness, 5) Domain adaptation techniques.
